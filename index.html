~$0.003/msg

~$0.02/scan

Using Claude Code efficiently
Tips to make sessions faster and cheaper.
Which model to use
Opus — Complex builds, big refactors, audits, multi-step tasks. Best for Baker Hub work.
Sonnet — Feature building, bug fixes, moderate complexity. Good balance.
Haiku — Quick simple changes (rename a label, change a colour). Cheapest and fastest.
Be specific to save cost
Instead of "fix the bins" say "in renderThisWeek() around line 1475, change Wednesday to Thursday".

index.html has 3 script blocks:
1. Main app — dashboard, NDIS, insurance, contacts, dates, todos, habits, animals, kids, house projects
2. Baker AI popup — starts after the Baker AI Chat Popup comment
3. AisleMate shopping — starts with AISLEMATE heading

Tell Claude which block to work in. Say "only change X, don't touch anything else."
Session management
Start fresh sessions when switching topics — long sessions slow down.
Reference CLAUDE.md — say "as documented in CLAUDE.md" to point to existing context.
Update CLAUDE.md at the end of each session so the next one has full context.
If You Move Away From Claude
App keeps running — The code is yours on GitHub, data is yours in Supabase. Nothing stops working.
Baker AI chat stops — The only thing that breaks. Update the Supabase Edge Function to point to OpenAI/Gemini/etc instead.
Receipt scanning stops — Same fix: update the Edge Function to use a different vision API.
Rename CLAUDE.md — It's just a text file. Rename to PROJECT.md or README-dev.md. The content is provider-agnostic.
Future development — Any developer or AI coding tool (Copilot, Cursor, etc.) can work on the code. It's standard HTML/CSS/JS.
